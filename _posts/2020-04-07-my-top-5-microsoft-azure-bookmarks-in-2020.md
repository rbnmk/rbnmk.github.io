I recently cleaned up my bookmarks since I started using the new Microsoft Edge. I also figured out I just need a small set of bookmarks for my day to day work. I use these websites on a daily basis for designing, implementing, configuring and securing Microsoft Azure infrastructure. Governance and fundamental stuff is a hot topic these days, which is available in the Microsoft cloud adoption framework. You&#8217;ve probably been to these pages before but I still feel like I can share the value these pages have.

  1. [Azure Resource Manager reference](https://docs.microsoft.com/en-us/azure/templates/): Since I am working with ARM templates and I obviously can&#8217;t remember every property of every resource.
  2. [Azure Resource Manager functions](https://docs.microsoft.com/en-us/azure/azure-resource-manager/templates/template-functions): Some ARM templates need logic. Also can&#8217;t remember every single one of them and how to use them.
  3. [Cloud Adoption Framework](https://docs.microsoft.com/en-us/azure/cloud-adoption-framework/): This website is essentially a big &#8220;how do I start and use Azure&#8221; guide. I use this to stay up to date about several best practices and recommended architectures.
  4. [Azure Pipelines reference](https://docs.microsoft.com/en-us/azure/devops/pipelines/get-started/pipelines-get-started?view=azure-devops): I recently started writing my templates as code. This page helps me a lot by providing examples and clear guidance how and when to use certain tasks for example.
  5. [KQL Reference](https://docs.microsoft.com/en-us/azure/kusto/query/): Since almost everything can log to Log Analytics now I often need to query to get certain results. 

There are some more pages that I use frequently (Google! sorry Bing!) but I find myself going to these pages directly and search from there.

If you have any recommendations for me let me know.
